معرفی

جاوا اسکریپت یک زبان برنامه نویسی متقاطع و شی گرا است. یک زبان کوچک و سبک است. در داخل یک محیط میزبان (به عنوان مثال مرورگر وب) ، JavaScript می تواند به اشیاء محیط آن وصل شود تا کنترل برنامه ای بر روی آنها انجام شود.

جاوا اسکریپت شامل یک کتابخانه استاندارد از اشیاء ، مانند Array ، Date و Math و مجموعه اصلی عناصر زبانی مانند اپراتورها ، ساختارهای کنترل و جمله ها است. هسته اصلی JavaScript را می توان با اهداف مختلف با افزودن آن با اشیاء اضافی ، تمدید کرد. مثلا:

  • JavaScript سمت مشتری با تهیه اشیاء برای کنترل یک مرورگر و مدل شیء اسناد (DOM) زبان اصلی را گسترش می دهد. به عنوان مثال ، برنامه های افزودنی سمت مشتری اجازه می دهد عناصر را بر روی فرم HTML قرار دهند و به رویدادهای کاربر مانند کلیک ماوس ، ورودی فرم و ناوبری صفحه پاسخ دهند.
  • سمت سرور JavaScript با تهیه اشیاء مربوط به اجرای JavaScript روی یک سرور ، زبان اصلی را گسترش می دهد. به عنوان مثال ، پسوندهای سمت سرور به یک برنامه کاربردی اجازه می دهد تا با یک پایگاه داده ارتباط برقرار کند ، تداوم اطلاعات را از یک فراخوان به برنامه دیگر یا ارائه دستکاری پرونده بر روی سرور ارائه دهد.
آنچه باید از قبل بدانید

این راهنما فرض می کند که شما زمینه اصلی زیر را دارید:

  • درک کلی از اینترنت و شبکه جهانی وب (WWW).
  • دانش خوب در مورد زبان نشانه گذاری HyperText (HTML).
  • برخی از تجربه برنامه نویسی. اگر تازه وارد برنامه نویسی هستید ، یکی از آموزشهای مرتبط با صفحه اصلی درباره JavaScript را امتحان کنید.
جاوا و جاوااسکریپت

جاوا اسکریپت و جاوا از بعضی جهات مشابه هستند اما در برخی دیگر اساساً متفاوت هستند. زبان جاوا اسکریپت شباهت زیادی به جاوا دارد اما دارای تایپ استاتیک جاوا و بررسی نوع قوی نیست. جاوا اسکریپت بیشترین موارد را دنبال می کند نحو بیان جاوا ، نامگذاری کنوانسیون ها و سازه های اصلی کنترل جریان که به همین دلیل تغییر نام آن از LiveScript به جاوا اسکریپت

برخلاف سیستم کامپایل زمان کلاس جاوا که توسط اعلانات ساخته شده است ، JavaScript از یک سیستم زمان اجرا بر اساس تعداد کمی از انواع داده ها که مقادیر عددی ، بولی و رشته را پشتیبانی می کنند ، پشتیبانی می کند. جاوا اسکریپت به جای رایج ترین مدل شیء مبتنی بر کلاس ، از یک شیء مبتنی بر نمونه اولیه برخوردار است. مدل مبتنی بر نمونه اولیه پویایی را فراهم می کند وراثت؛ یعنی آنچه به ارث رسیده می تواند برای اشیاء فردی متفاوت باشد. جاوا اسکریپت همچنین از توابع بدون اعلامیه ویژه پشتیبانی می کند الزامات. توابع می توانند خصوصیات اشیاء باشند روشهای گسست

JavaScript در مقایسه با جاوا یک زبان بسیار آزاد است. شما نمی کنید باید تمام متغیرها ، کلاس ها و روش ها را اعلام کند. لازم نیست نگران این باشید که روش ها عمومی ، خصوصی یا محافظت شده باشد و نیازی به اجرای رابط ها نیست. متغیرها ، پارامترها و انواع بازده عملکرد به صراحت تایپ نمی شوند.

برای شروع کار با نوشتن JavaScript ، Scratchpad را باز کرده و اولین کد JavaScript "سلام جهان" را بنویسید: function greetMe(yourName) { alert("Hello " + yourName); } greetMe("World"); كد موجود در پد را انتخاب كنيد و Ctrl + R را بزنيد تا در مرورگر شما فاش شود!
متغیرها

شما از متغیرها به عنوان نامهای نمادین برای مقادیر در برنامه استفاده می کنید. نام متغیرها ، به نام شناسه ها ، مطابق با قوانین خاص است.

شناسه جاوا اسکریپت باید با یک نامه ، تأکید (_) یا علامت دلار ($) شروع شود. شخصیت های بعدی همچنین می توانند به رقم (0-9) باشند. زیرا جاوا اسکریپت حساس به مورد است ، حروف شامل شخصیت های "A" از طریق "Z" (حروف بزرگ) و شخصیت های "a" از طریق "z" (حروف کوچک) است.

می توانید از شناسه های ISO 8859-1 یا Unicode مانند å و use استفاده کنید. همچنین می توانید از دنباله های فرار یونیکد به عنوان کاراکتر در استفاده کنید شناسه ها برخی از نمونه های اسامی قانونی Number_hits ، temp99 و  _نام.

اعلام متغیرها
شما می توانید یک متغیر را از سه طریق اعلام کنید:

با کلمه کلیدی var. به عنوان مثال ، var x = 42. از این نحو                 می توان برای اعلام متغیرهای محلی و جهانی استفاده کرد.

با اختصاص دادن یک مقدار. به عنوان مثال ، x = 42. این همیشه متغیر جهانی را اعلام می کند. این یک اخطار شدید JavaScript ایجاد می کند. شما نباید از این نوع استفاده کنید.

با کلمه کلیدی let. به عنوان مثال ، let y = 13. از این نحو  می توان برای اعلام متغیر محلی دامنه بلوک استفاده کرد. دامنه متغیر را در زیر مشاهده کنید.

دامنه متغیر

وقتی متغیر را خارج از هر عملکردی اعلام می کنید ، به آن "a" گفته می شود متغیر جهانی ، زیرا در دسترس است با کد دیگری در سند فعلی وقتی متغیر را در یک تابع اعلام می کنید ، آن را متغیر محلی می نامند ، زیرا فقط در آن موجود است تابع.

JavaScript قبل از ECMAScript 2015 دامنه بیانیه بلوک ندارد. بلکه ، یک متغیر اعلام شده در یک بلوک برای عملکرد محلی است (یا دامنه جهانی) که این بلوک در آن قرار دارد. برای مثال موارد زیر کد 5 وارد خواهد شد زیرا دامنه x تابعی است (یا جهانی) متن) که در آن x اعلام می شود ، نه بلوکی که در این حالت عبارت if است.

if (true) { var x = 5; } console.log(x); // 5

این رفتار هنگام استفاده از اعلامیه اجازه در ECMAScript 2015 تغییر می کند.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
متغیرهای سراسری

متغیرهای جهانی در حقیقت خاصیت های هدف جهانی هستند. در وب صفحه هایی که شیء جهانی پنجره است ، بنابراین می توانید جهانی تنظیم کرده و به آن دسترسی داشته باشید متغیرها با استفاده از syntax windows.variable.

در نتیجه ، می توانید به متغیرهای جهانی اعلام شده در یک پنجره یا با مشخص کردن نام پنجره یا قاب ، از پنجره یا قاب دیگر فریم کنید. به عنوان مثال ، اگر متغیری به نام phoneNumber در یک سند اعلام شده باشد ، می توانید از یک iframe به عنوان mother.phoneNumber به این متغیر مراجعه کنید.

ثابت ها

با کلمه کلیدی const می توانید فقط خواندنی ایجاد کنید. نحو یک شناسه ثابت همانند متغیر است شناسه: باید با یک حرف ، علامت تأکید یا دلار شروع شود و می تواند دارای حروف الفبا ، عددی یا زیربنایی باشد.

const PI = 3.14;

ثابت نمی تواند مقدار را از طریق واگذاری تغییر دهد یا دوباره اعلام شود در حالی که فیلمنامه در حال اجرا است. این باید به یک مقدار اولیه شود.

قوانین دامنه ثابت برای قوانین ثابت است متغیرها اگر کلمه کلیدی const نادیده گرفته شود ، شناسه فرض می شود یک متغیر را نشان می دهد.

شما نمی توانید ثابت را با همان اسم یک تابع یا متغیر در همان دامنه. مثلا:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements } با این حال ، از ویژگی های شی محافظت نمی شود ، بنابراین جمله زیر  بدون مشکل اعدام شده است const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";
Data types

The latest ECMAScript standard defines seven data types:

  • Six data types that are primitives:

    • Boolean. true and false.
    • null. A special keyword denoting a null value. Because JavaScript is case-sensitive, null is not the same as Null, NULL, or any other variant.
    • undefined. A top-level property whose value is undefined.
    • Number. 42 or 3.14159.
    • String. "Howdy"
    • Symbol (new in ECMAScript 2015). A data type whose instances are unique and immutable.
  • and Object
Although these data types are a relatively small amount, they enable you to perform useful functions with your applications. Objects and functions are the other fundamental elements in the language. You can think of objects as named containers for values, and functions as procedures that your application can perform.
if...else statement
Use the if statement to execute a statement if a logical condition is true. Use the optional else clause to execute a statement if the condition is false. An if statement looks as follows: if (condition) { statement_1; } else { statement_2; } condition can be any expression that evaluates to true or false. See Boolean for an explanation of what evaluates to true and false. If condition evaluates to true, statement_1 is executed; otherwise, statement_2 is executed. statement_1 and statement_2 can be any statement, including further nested if statements.

You may also compound the statements using else if to have multiple conditions tested in sequence, as follows:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } In the case of multiple conditions only the first logical condition which evaluates to true will be executed. To execute multiple statements, group them within a block statement ({ ... }) . In general, it's good practice to always use block statements, especially when nesting if statements: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } It is advisable to not use simple assignments in a conditional expression, because the assignment can be confused with equality when glancing over the code. For example, do not use the following code: if (x = y) { /* statements here */ } If you need to use an assignment in a conditional expression, a common practice is to put additional parentheses around the assignment. For example: if ((x = y)) { /* statements here */ }
while statement
A while statement executes its statements as long as a specified condition evaluates to true. A while statement looks as follows: while (condition) statement If the condition becomes false, statement within the loop stops executing and control passes to the statement following the loop.

The condition test occurs before statement in the loop is executed. If the condition returns true, statement is executed and the condition is tested again. If the condition returns false, execution stops and control is passed to the statement following while.

To execute multiple statements, use a block statement ({ ... }) to group those statements.

Example:

The following while loop iterates as long as n is less than three:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

With each iteration, the loop increments n and adds that value to x. Therefore, x and n take on the following values:

  • After the first pass: n = 1 and x = 1
  • After the second pass: n = 2 and x = 3
  • After the third pass: n = 3 and x = 6

After completing the third pass, the condition n < 3 is no longer true, so the loop terminates.

Function declarations
A function definition (also called a function declaration, or function statement) consists of the function keyword, followed by:
  • The name of the function.
  • A list of arguments to the function, enclosed in parentheses and separated by commas.
  • The JavaScript statements that define the function, enclosed in curly brackets, { }.

For example, the following code defines a simple function named square:

function square(number) { return number * number; }

The function square takes one argument, called number. The function consists of one statement that says to return the argument of the function (that is, number) multiplied by itself. The return statement specifies the value returned by the function.

return number * number;

Primitive parameters (such as a number) are passed to functions by value; the value is passed to the function, but if the function changes the value of the parameter, this change is not reflected globally or in the calling function.
